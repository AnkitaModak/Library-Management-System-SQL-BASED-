# 📚 Library Management System

## Overview

The Library Management System is a SQL-based database project designed to manage the day-to-day operations of a library. It stores and organizes information about members, books, authors, publishers, borrowing records, and book returns. The project demonstrates the use of SQL for creating and managing relational databases.

## Features

* Member registration and management
* Book catalog management
* Author and publisher records
* Issue and return of books
* Track book availability
* Maintain borrowing history
* Generate reports using SQL queries

## Technologies Used

* Oracle SQL
* Oracle SQL Developer

## Database Tables

* **MEMBERS** – Stores member details.
* **BOOKS** – Stores book information.
* **AUTHORS** – Stores author details.
* **PUBLISHERS** – Stores publisher information.
* **BOOK_COPIES** – Tracks individual copies of books.
* **ISSUE_RECORDS** – Maintains book issue information.
* **RETURN_RECORDS** – Maintains return information.

## Learning Objectives

This project helps practice:

* DDL (CREATE, ALTER, DROP, TRUNCATE)
* DML (INSERT, UPDATE, DELETE)
* Constraints (PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, NOT NULL)
* Joins
* Aggregate Functions
* GROUP BY and HAVING
* Subqueries
* Views
* Indexes
* Transactions (COMMIT, ROLLBACK, SAVEPOINT)

## How to Run

1. Open Oracle SQL Developer.
2. Create a new database connection.
3. Execute the table creation scripts.
4. Insert sample data into the tables.
5. Run SQL queries to perform library operations and generate reports.

## Sample Operations

* Add a new member
* Add new books
* Search for books
* Issue a book
* Return a book
* Check available books
* View borrowing history
* List overdue books
